# アンダーグラウンド 文体ガイド v1.0

日付: 2026-07-19 ／ 根拠: 実テキスト30本（6系統×5）の分析 ／ 用途: 全テキスト量産の正典

## 原則5条

1. **短く刺す**: 台詞は2文以内。ログ・イベント文も2文構成が基本。
2. **効果は物語で語り、数値は語らない**: 「次の出演通知が来やすくなる」○／「通知率+10%」×。
3. **敬語はチャネルで固定**: バンド間・打ち上げ＝タメ口。運営・ファン・事務メール＝です・ます。混用禁止。
4. **モチーフ語彙圏を回す**: 下表の語彙が世界の声。新規テキストは最低1語をこの圏から取る。
5. **主人公は寡黙**: 短い受け・問い返し（「丸い？」型）のみ。3文以上の主張・演説は禁止。

## モチーフ語彙表

中核: 残る／削る／鳴る／刺さる／届く／地下／名前（を置く・が近づく）／荒い（未完成の褒め言葉）。※「床」は退役（使用しない）。
準核: 尖る／磨く／逃げ道／本気／現場。
**NG語彙**: キラキラ／夢を叶える系の直接表現／ビジネス用語（コンテンツ・バズる等）／数値・％の直接言及（UI表示は除く）。

## 系統別テンプレ

### 系統1 ノベル会話
構造: 一文目で相手の状態を突く→二文目で誘い・示唆。音楽観＝人生観の比喩。
既存例: ナキリ「削れよ。傷つくくらい尖った音の方が、残る時もある。」
新規例: ミケ「フェスの影って言われてさ。影ってことは、光の近くにいるんだよ。」

### 系統2 代表者コメント・裏話（『』）
構造: 一文目で常識を否定or転回→二文目で自分の流儀。金言形式。
既存例: ラルゴ『憧れは遠くに置くな。届かない場所に置いた瞬間、言い訳になる。』
新規例: ダイドウジ『重い音は威嚇じゃない。逃げ場を塞いで、正面から聴かせるためだ。』

### 系統3 イベントログ
構造: 主語省略・過去形。一文目＝出来事、二文目＝関係や状況の変化。感情語なし。
既存例: 「CARBONSとの対バン後、ナキリから声をかけられた。」
新規例: 「SHELTERとの対バンが決まった。クリオから短い返信が来た。」

### 系統4 SNS投稿
構造: 1文の断言。バンドの哲学を凝縮。ハッシュタグ・絵文字なし。
既存例: magnet_wolf「綺麗に鳴らすより、刺さる音を鳴らせ。」
新規例: nova_biscuit「宇宙まで届かなくていい。隣の駅まで鳴れば、今日は勝ちだ。」

### 系統5 メール
構造: 改行のリズムで話し言葉を運ぶ。署名は「バンド名　名前」（全角スペース）。運営は疑問形の柔らかい勧誘。ファンは具体曲名参照＋二文。
既存例: 「はじめまして。\nライブハウスUNDERの運営です。（略）よければ、出演しませんか？」
新規例（ファン）: 「対バンで初めて観ました。\n${1曲目}のイントロ、まだ耳に残ってます。」

### 系統6 打ち上げ選択肢イベント文
構造: 一文目＝出来事（過去形）、二文目＝効果の物語的言い換え（原則2の適用箇所）。
既存例: 「${共演者}と連絡先を交換した。次の出演通知が来やすくなる。」
新規例: 「機材の運び方ひとつで場が締まると教わった。現場の目が少し変わる。」

## キャラ声レジスタ（判明分・T1描画リストと連動）

タカナシ＝兄貴分（〜だぜ・！可）／ナキリ＝鋭利・命令形／クリオ＝静かな強さ・守る語彙／モズ＝脱力・ゆるさ／ミケ＝陽転換／ダイドウジ＝重量・正面／カエデ＝言葉への執着／ラルゴ＝王者の説教／ヒノカワ＝丁寧語（「んです」調・静・和）／店長・運営＝です・ます固定。未確立キャラは中立文体＋所属バンドのlore気質から導出。

## 記号・書式規則

台詞「」／代表者コメント『』／絵文字・顔文字なし／「！」は熱量キャラ（タカナシ等）のみ／メール改行は\n・署名全角スペース／変数差し込みは既存${}形式を踏襲。

## 量産運用（AI委託時）

1発注＝1系統×N本。委託文には本ガイド＋対象バンドのlore・voice・タグを添付。受け入れチェック: 文数上限／チャネル敬語／モチーフ語彙1語以上／NG語彙ゼロ／主人公寡黙則。
**声の優先順位**: レジスタ表に未記載のキャラは、既存台詞・既存loreの口調を一次ソースとして踏襲する（表はデータの要約であり、データより優先されない）。既存と矛盾する解釈が必要な場合のみ発注者へ照会。

## SNS細則（v1.1追補・TEXT-2A裁定より）

SNSは名言・金言の置き場ではない。**Xのバンドマンの生活タイムライン**として書く。告知・エゴサ反応・雑談・対バン礼賛・自虐が混在する密度が正義で、支離滅裂でも生活感があればよい。「床」は使用しない（語彙圏から退役）。金言調・説教調はSNSでは禁止（それはlore/会話の役割）。

バンドマンつぶやきの型（各バンド固定ボイスは3〜5個をこの型から配分）:
- **告知**: 「○/○ ○○（ハコ名）出ます。来て」系。素っ気なくてよい。
- **エゴサ/反応**: 「○○（対バン名）との企画、めちゃくちゃかき鳴らした」「来てくれてありがとう、俺たちのホーム最高」系。
- **生活/雑**: 「新曲思いついた」「新作フラッペうまい」「ラーメン食った」系。バンドと無関係でよい。
- **対バン/シーン**: 「○○（他バンド名）最高」「新しいバンド見てると懐かしくなる」系。
- **自虐/弱音**: 「音源全然進まん」「またチケット手売りの季節」系。
バンドの性格は語彙と温度で出す（ナキリは短く刺す/モズは脱力/タカナシは熱い）が、**全員が四六時中バンド哲学を呟くわけではない**ことを守る。

## ライブハウス名（新設・世界の実在感）

ハコは「駅裏のライブハウス」でなく固有名で呼ぶ。名称一覧を別ファイルで管理し、SNS・ブッキング・メールで共用する。命名の質感: 地下・DIY・無骨（例の方向性はVENUE一覧で定義）。

## 会場の呼び方（話者別・v1.2）

同じ会場でも話者で語彙が変わる。混同しないこと。
- **演者（バンドマン一人称＝SNS・打ち上げ・バンド間会話）**: 自分の出番を「ライブ」「企画」「ツアー」「ワンマン」「対バン」と呼ぶ。会場名は固有名（VENUE_NAMES）で直に呼ぶ。「ハコ」は演者はあまり使わない。
- **客・裏方・システム（ブッキング表記・世界反応・運営メール）**: 「ハコ」「会場」「小箱/中箱/大箱」を使ってよい。
- 例: SNS告知「今度の企画、○○（VENUE）で一本やる」／ブッキング画面「小箱・キャパ55」。
