# AUDIT PACKET v0.3.67-ui-support-growth-tuning-draft

## 監査目的

v0.3.66-balance-draft-fix1からv0.3.67で行ったUI・サポート・成長体感調整について、コード欠陥と体感調整を分けて確認する。

## 比較元

- v0.3.66-balance-draft-fix1

## 監査対象

- v0.3.67-ui-support-growth-tuning-draft

## 重点監査項目

### 1. 差分・変更範囲

- 汎用サポート削除がライブ計算を壊していないか。
- 名前付きサポート11人化で参照欠けがないか。
- メールモーダル化で既存返信導線が壊れていないか。
- ライブ準備チェックが既存ライブ準備に干渉していないか。
- 成長補正の数値変更が大きすぎないか。

### 2. 変更禁止領域

以下に退行がないか確認。

- GRAND条件。
- ターン進行ルール。
- 作詞/作曲/曲完成/曲強化でターン進行しない仕様。
- 通常行動/ライブ/打ち上げ/ドタキャンのみターン進行対象。
- メールからのライブ参加返信導線。
- メールからのメンバー候補返信導線。
- DEVモード。
- コピー曲ガード。

### 3. 旧セーブ互換

- `livePrepSupportIds` に `sup_guitar` などの旧汎用サポートIDが残っていても壊れないか。
- `supportAffinity` に旧汎用サポートIDが残っていても壊れないか。
- growthType未設定メンバーが引き続き正規化されるか。

### 4. サポート契約

- 汎用サポートが表示されないか。
- 既存8人＋新規3人、合計11人が表示されるか。
- リオ/ナツメ/ミツキがライブ計算上のサポートとして扱われるか。
- サポート費用が利益計算に反映されるか。
- supportAffinityが名前付きサポートIDで加算されるか。

### 5. メールモーダル

- 件名タップで本文モーダルが開くか。
- 閉じるボタンで戻れるか。
- ライブ参加返信がモーダル内からできるか。
- メンバー候補返信がモーダル内からできるか。
- 予定表確認/候補一覧遷移でactiveMailIdが残って邪魔しないか。

### 6. ライブ準備チェック

- 基本パート不足警告が出るか。
- セトリ5曲チェックが出るか。
- コピー曲多め警告が出るか。
- 疲労/資金警告が出るか。
- 警告があってもライブ開始できるか。

### 7. 成長タイプ調整

- センス型/カリスマ型の重点成長が改善しているか。
- 序盤型が後半も強すぎないか。
- 晩成型が後半に伸びるか。
- 総量差が大きく開きすぎていないか。

## 期待結果

- コード欠陥がなければfix不要。
- メール/サポート/ライブ準備のスマホ体感は実機確認へ回す。
- 成長タイプ数値は仮決定のため、Fable5または後続シミュレーションで再評価する。
