# CHANGELOG v0.3.67-ui-support-growth-tuning-draft

## 目的

v0.3.66-balance-draft-fix1を土台に、スマホ実機で気になりやすいUI導線と体感差を先に整える版。
主にライブ準備、サポート契約、メール閲覧、成長タイプ体感を調整した。

## 変更内容

### 1. 汎用サポート廃止

ライブ準備画面で表示されていた以下の汎用サポートを廃止。

- サポートギター
- サポートベース
- サポートドラム
- サポートキーボード
- サポートDJ
- その他楽器サポート

今後のサポート契約は、名前付きサポートのみを選ぶ形式に統一。
旧セーブで `sup_guitar` などの汎用サポートIDが `livePrepSupportIds` に残っている場合は、正規化時に破棄する。

### 2. 名前付きサポート3人追加

既存8人に加えて、特殊楽器の名前付きサポートを3人追加。

- リオ：Piano / Synth
- ナツメ：Sax / Brass
- ミツキ：Percussion

`other` という曖昧な枠はライブ準備のサポート契約から外し、具体楽器として扱う。

### 3. サポートUI整理

ライブ準備画面のサポート契約欄を「名前付きサポート契約」に変更。
表示情報を以下に整理。

- 名前
- 担当パート
- 契約費
- 効果値
- 相性値
- 得意ジャンル
- 補足メモ

不足している基本パートに合うサポートを上に出しやすくするため、表示順に簡易優先度を追加。

### 4. ライブ準備チェック追加

ライブ準備画面に「ライブ準備チェック」パネルを追加。

確認項目：

- 基本パート不足
- セトリ5曲
- コピー曲多め警告
- 疲労
- 資金
- サポート人数

警告があっても本番には進める。プレイヤーが準備不足に気づきやすくするためのガイド。

### 5. メール本文をモーダル化

メール一覧の下に本文を出す方式をやめ、件名タップで本文モーダルを開く方式に変更。

- 件名タップで本文モーダル表示
- 本文内に返信/予定確認/候補一覧ボタンを維持
- 閉じるボタンでメール一覧へ戻る
- メール一覧下部までスクロールする必要を減らす

### 6. 成長タイプ体感調整

Fable5監査提案をもとに、数値を仮調整。

- 練習時 focused growth：0.32 → 0.45
- 非Sライブ focused growth：0.45 → 0.55
- 序盤型 31T以降：0.98 → 0.90
- 晩成型 35T以降：1.14 → 1.20

狙い：

- センス型/カリスマ型の名前負けを軽減
- 序盤型が後半も総量トップになりすぎる問題を軽減
- 晩成型の後半伸びを体感しやすくする

## 維持仕様

- GRAND条件は変更なし。
- ターン進行ルールは変更なし。
- 作詞/作曲/曲完成/曲強化でターン進行しない仕様は維持。
- メールからのライブ参加返信/メンバー候補返信導線は維持。
- DEV GRAND条件チェックは維持。
- コピー曲ガードは変更なし。

- ライブ準備チェックの見やすさ。
- サポート11人表示時のスクロール量。
- 成長タイプ体感の長期バランス。
