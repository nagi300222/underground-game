# CHANGELOG v0.4.0-rc3

## 概要

v0.4.0-rc2の実機確認で出たUX不具合を最小修正した版。

新機能追加ではなく、実機操作でストレスになった2点と、スマホ可読性の安全パッチのみ。

## 修正

### 1. DEV details保持

DEV画面の折りたたみカテゴリが、DEV操作後の再描画で毎回初期状態に戻っていた。

- `devOpenGroups` を追加。
- `details.dev-group` のopen状態をtoggle時に保持。
- DEV操作で `render()` されても開閉状態を維持。

### 2. 携帯タブ押下時のトップ復帰

携帯内でSNS/メール/図鑑を開いた状態が残り、下部タブの「携帯」を押しても前回サブ画面から始まっていた。

- 5タブの「携帯」へ遷移する時は `phoneSubView = "menu"` に戻す。
- `activeMailId` もクリア。
- 携帯内部の「メール/SNS/図鑑」遷移は従来通り維持。

### 3. スマホ可読性の最小CSS補強

v0.4.1本格デザイン前の安全パッチとして、640px以下のみ以下を調整。

- カード内余白を少し増加。
- 本文/小文字の行間を増加。
- ボタンの最小高さを補強。
- 見出しサイズを微調整。

## 変更しないもの

- GRAND条件
- ターン進行
- ライブ準備DOM読取
- メール承認処理
- 図鑑解放条件
- SNS重複制御
- セーブ/ロード危険一時状態
