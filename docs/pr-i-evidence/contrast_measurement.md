# PR-I 条件解消: アクセント面上文字の実測コントラスト

対象: `.v042-songcraft`（作詞・作曲ボタン、ACCベタ面 `rgb(255,74,0)`）上のテキスト3箇所。
測定方法: Playwright実描画（device_scale_factor=4）でボタン単体をスクリーンショットし、
テキスト部分の最暗ピクセル平均と、テキストなし領域のベタ面ピクセルをサンプリング。
WCAG相対輝度式でコントラスト比を算出（`docs/pr-i-evidence/*_contrast_*.png` が実描画証跡）。

| 要素 | 内容 | 発見時の状態 | 実測コントラスト | 発見時の判定 | 修正後の状態 | 実測コントラスト | 修正後の判定 |
|---|---|---|---|---|---|---|---|
| `<small>`/`<em>` | 「今ターンの制作は完了」等の副文言・バッジ | `color:var(--bg)` + 要素`opacity:.72` | **4.16:1** | ✗ 4.5:1未満（違反） | `opacity:1` | **5.82:1** | ✓ |
| `b::before` | 英字ミニラベル「SONGCRAFT」 | `color:var(--bg)`（継承） + 要素`opacity:.7` | **3.97:1**（実描画未測定・計算値） | ✗ 4.5:1未満（違反） | `opacity:1` | **5.83:1**（実測） | ✓ |
| `<span>`/`<b>` | アイコン・見出し「作詞・作曲」 | `color:var(--bg)`、opacity指定なし（=1） | 5.83:1（計算値、修正不要） | ✓ | 変更なし | — | ✓ |

## 原因

`docs/SKIN_REDESIGN_POLICY.md` v2.1時点の文字色規則は「BG系ダーク（`rgba(11,11,16,.6以上)`）」という**色のalpha値**のみを規定しており、要素自体のCSS `opacity`プロパティによる実効レンダリング時の希釈を規定していなかった。本件の2箇所は色そのものは不透明な`var(--bg)`だったが、要素`opacity:.7`前後が別途適用されており、ACCベタ面（明るい橙）に対して合成された結果、実効コントラストが4.5:1を下回っていた。

PR-Iの当初の「全面点検・違反0件」は色のrgba値のみを走査した結果であり、要素opacityの希釈を見落としていた。今回の条件解消でこの2箇所を修正し、`docs/SKIN_REDESIGN_POLICY.md`の文字色規則を「rgba値かつ要素opacity込みの実測4.5:1以上」へ改訂（v2.2）。
