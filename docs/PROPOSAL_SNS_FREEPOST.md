# 提案書: SNS自由投稿機能（1日1回）

作成: TEXT-2A改 E項 ／ ステータス: 提案のみ・未実装 ／ 確定事項: ゲーム数値への影響なし・フレーバー専用（発注者確定済み）

## 背景

委託文TEXT-2A改E項により、プレイヤーがSNSへ自由文を1ターン1回投稿できる機能の実装が指示された。ただし本項目のみ「UI提案を先行提出し、実装は発注者確認後」と明記されているため、本書は設計提案のみを行い、コードへの実装は行っていない。

## 提案内容

### 1. 入力UIの置き場所

**案**: 携帯のSNSタブ（`renderSnsScreen`が描画する画面）の投稿一覧の直上に、既存の「告知」「雑談」ボタン（`snsPost(kind)`）と並べて「自由投稿」ボタンを追加する。タップすると、テキストエリア＋文字数カウンタ＋投稿ボタンを持つ小さな入力オーバーレイ（既存の`event-modal`系オーバーレイと同じ意匠）を開く。

**代替案**: 既存の「告知」「雑談」ボタンの並びに3つ目の選択肢として統合し、`snsPost("free")`のような分岐を追加する（UIの新規追加を最小化できる一方、自由文である以上テキスト入力欄が別途必要なため、結局オーバーレイは要る）。

いずれの案でも、既存の「SNS投稿は1日1回まで」という`state.snsLastPostTurn`の制限ロジックはそのまま流用できる（告知・雑談・自由投稿を合わせて1日1回、という既存制限に自由投稿も含める想定）。

### 2. 文字数上限

**案: 140字**。X（旧Twitter）を参照した実在感重視の数値。既存の固定ボイス・世界反応投稿が概ね20〜60字程度であることを踏まえると、140字は「多少長く書いても許容される」実用的な上限になる。バリデーションは入力オーバーレイ側でリアルタイムにカウンタ表示し、上限超過時は投稿ボタンを無効化する形を想定。

### 3. 既存タイムラインへの差し込み方

**案**: 既存の`addSnsPost(author, body, mood, bandId)`をそのまま流用し、`author`は`"@our_band"`（既存の自バンド投稿と同じハンドル）、`mood`は新規の`"free"`を割り当てる。これにより既存の`state.snsPosts`配列に時系列で自然に混在し、`renderSnsScreen`側の表示ロジックも無変更で対応できる（`mood`によるスタイル分岐が既存にあれば、"free"用の見た目を1つ追加する程度で済む想定）。

### 4. 不適切文対策の要否

ローカル動作・オフライン・自分しか見ない投稿であるため、技術的な不適切文フィルタは**不要と考えられる**（外部公開やマルチプレイ要素が無いため、悪用の実害がない）。ただし以下の軽微な配慮は検討の余地あり:
- 空文字・空白のみの投稿を弾く（最小バリデーション）
- 改行を許可するか（1行のみ推奨、他のSNS投稿との見た目の一貫性のため）

**発注者確認をお願いしたい点**: 上記の「対策不要」判断でよいか、それとも何らかの最小限のガード（NGワードリスト等）を望まれるか。

## 確定事項（委託文より）

- 1ターン1回まで（既存のSNS投稿回数制限の枠内）。
- 内容は自由（支離滅裂でも可）。
