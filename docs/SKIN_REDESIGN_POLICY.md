# スキン刷新方針 v1.2（STUDY 06確定）

作成: 2026-07-17 ／ ステータス: 確定 ／ 置き場所: `docs/`（PR-Cで上書き）
改訂履歴: v1.1→v1.2 実機適用で判明した可読性事故（説明文の橙化）を受け、機能色拡張・ピクト体系追加・裁定②③改訂・現場翻訳（STUDY 06）反映

## 位置づけ

v0.4.3スコープ外の正式版後課題。段階1（PR #12: トークン基盤・フォント・監査ツール）は適用済み前提。本版はPR-C（CSS限定のv1.2適用）とPR-B改（game.js表層: ピクト置換）で完成させる。凍結事項には一切干渉しない。

## 診断（追補）

v1.1の敗因: 二色経済では「元々色付きだった文字」の写像先がACCしかなく、説明文が橙の連続文になった。教訓として**文字色の絶対規則**と**密度のある画面での事前検証**を制度化する。

## トークン（v1.2）

| 役割 | 値 | 用途 |
|---|---|---|
| BG | `#0B0B0D` | 全面基調 |
| SURFACE | `#14120E` | パネル背景。影なしの奥行き1段 |
| INK | `#EAE6DD` | 読ませる文字すべて・罫線・非活性枠 |
| DIM | `#948C7C` | ラベル・補足（v1.1の#6E695Fから引上げ） |
| RULE | `#33312C` | 弱い区切り罫 |
| ACC | `#FF4A00` | 導線・ブランド専用（連続文禁止） |
| YEL | `#FFE600` | 強調マーカー・資金系数値（対BG約15:1） |
| RED | `#FF3B30` | 機能色: 通知バッジ・警戒・エラー |

## 文字色の絶対規則

読ませる文字は必ずINK。ACCの連続文・説明文・本文使用は全面禁止。強調はYEL＋下線2pxのマーカー方式。補足はDIM。無効・選択不可は低輝度ACC（`rgba(255,74,0,.32〜.4)`）＝「読ませない場所」限定。

## 組版・形状

角丸ゼロ、枠2px（弱区切り1px）、コンデンス大文字＋広字間、影・グラデ禁止は継続。ゲージは**VUセグメント方式**: fill＝`repeating-linear-gradient(90deg, 色 0 5px, transparent 5px 7px)`。疲労トラックには閾値位置に赤ゾーン目盛。打診・イベント系パネルは**半券方式**: SURFACE地＋左ミシン目（2px dashed DIM）＋上下切欠き円（擬似要素・DOM不変）。

## 発光・機能色規則

発光は導線2箇所固定（主ボタン: ACC枠＋文字＋内外グロー／活性タブ: 文字＋グロー＋`▌`）。ACCベタ塗り面は禁止継続。**例外**: REDの通知バッジ小円のみベタ可（万国共通記号のため）。疲労は閾値超過で数値・fillをRED化（クリップ表現。旧裁定③のACC枠案を置換）。

## ピクト体系（裁定②改訂: 自作ピクト主・テキスト記号補助）

作図規則: 24pxグリッド／線幅2px固定／`stroke-linecap:square`・miter／角度0°・45°・90°のみ／曲線は円だけ／主要な閉形にステンシル欠き1箇所／塗りは状態表示の小面のみ／`currentColor`継承／インラインSVG（外部参照禁止）。
確定9種: ライブ（45°ボーカルマイク・グリルクロス）／練習（コンボアンプ）／作詞作曲（連桁八分・菱形符頭）／人気（腕の上がったフロア）／資金（¥コイン）／疲労（バッテリー）／打診（封筒）／記録（カセット）／ホーム（ガレージ扉）。コードはSKIN_ORDER_v2付録B。不足分は同規則で追加し発注者判定。テキスト記号を補助で使う場合はU+FE0E必須。

## 見せ場原則

基調は禁欲を維持し、T30・T50フェスとライブ結果画面のみ演出強化の例外枠（多色可）。50ターンの単調さを断つための構造。具体演出はコンテンツ拡充側で設計。

## ネガティブ仕様

グラデ／角丸／ドロップシャドウ／絵文字／紙・ノイズ質感／走査線／等幅計器数字／ACCベタ面／**橙の連続文**。

## 適用範囲と実装リスク

CSS変更はPR-C、game.js表層はPR-B改。game.jsの同名関数罠は継続——段階1で実証済み（renderNav・renderHomeScreenが各2定義、死んだ旧実装のセレクタに発光を当てる事故が実際に発生、実描画確認で検知）。**編集・セレクタ設計は必ず実描画確認とセット**とする。
