# 監査パケット v0.3.70-band-events-extension-draft

## 監査目的

v0.3.70で追加した他バンド短編イベントについて、参照欠け・重複発火・優先度・持ち越し・進行不能がないか確認する。
同時に、v0.3.67〜v0.3.69のUI/打ち上げ/通常行動イベントが退行していないか確認する。

## 重点確認

### 1. 差分・変更範囲

- `BAND_EVENT_DATABASE` に7イベント追加。
- `BAND_EVENT_STORY_MAP` に7シーン追加。
- `STORY_SCENE_DATABASE` に7シーン追加。
- VERSION/SAVE_VERSION/title/cacheをv0.3.70へ更新。

### 2. 変更禁止領域

以下に意図せず触れていないか確認。

- GRAND条件。
- ターン進行ルール。
- コピー曲ガード。
- メール返信導線。
- DEV既存補助。
- ライブ結果処理。
- 打ち上げ後の次ターン進行。

### 3. イベント/ノベル

- 追加イベントの `bandId` が存在すること。
- `eventSeen` 条件の参照元イベントが存在すること。
- `lastBattleBandId` のバンドIDが存在すること。
- `BAND_EVENT_STORY_MAP` のイベントID/シーンIDに参照欠けがないこと。
- 追加7イベントが実プレイ相当条件で発火すること。
- 発火後、ノベル完走できること。
- 1ターン1件制御が維持されること。
- 既存の高優先イベントが残っている場合は、持ち越しになることをバグ扱いしない。

### 4. 回帰確認

- 名前付きサポート11人維持。
- 汎用 `sup_` サポートが復活していないこと。
- メール本文モーダル維持。
- 休憩ミニイベント発生率0維持。
- 打ち上げイベント維持。
- `pendingAfterpartyEvent` が保存から除外されること。
- DEV GRAND最低値セットが動くこと。

## 実行済みコード確認

- `node --check game.js`
- `node audit70_smoke.js`
- `node audit70_regression.js`

## 既知の注意

監査中、JACK BOMB/HYPER/ultimate quokkas/RUMBLE/Neon Reefの追加イベントが一見未発火に見えるケースがあった。
原因は、既存SHELTER/KAEDE/LACTイベントが未消化で、優先度的に先に発火していたため。
該当前提イベントを消化済みにした実プレイ相当条件では、追加7イベントすべて正常発火・完走を確認済み。
